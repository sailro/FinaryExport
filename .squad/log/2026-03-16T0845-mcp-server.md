## Key Decisions
1. **Core namespace strategy:** RootNamespace=FinaryExport prevents cascade impact (CLI + MCP both import as FinaryExport.*)
2. **Session-only auth:** Per user directive — MCP cannot cold-auth, must reuse CLI session
3. **All read-only:** Zero mutations exposed (future phase can add if needed)
4. **Bootstrap required:** All data tools depend on `finary_get_org_context` state setup
