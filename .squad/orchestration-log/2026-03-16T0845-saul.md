Complete MCP server with 15 tools across 7 tool classes:

   - **PortfolioTools:** `finary_get_portfolio_summary`
   - **AccountTools:** `finary_get_all_accounts`, `finary_get_accounts_by_category`, `finary_get_account_timeseries`
   - **TransactionTools:** `finary_get_all_transactions`, `finary_get_transactions_by_category`, `finary_get_transaction_categories`

   - **AllocationTools:** `finary_get_portfolio_allocations`, `finary_get_allocation_distribution`, `finary_get_allocation_fees`

3. **Architecture:**
   - `Program.cs` — `Host.CreateApplicationBuilder` + `AddFinaryCore()` + `AddMcpServer().WithStdioServerTransport().WithToolsFromAssembly()`
   - Console logging redirected to stderr (stdout reserved for MCP protocol)
   - Primary constructor injection with `IFinaryApiClient` in all tool classes
   - Category enum parsing shared via `AccountTools.ParseCategory()`

4. **Auth & session:**
   - Per user directive: session-only auth via `session.dat` from CLI
   - `McpCredentialPrompt` throws `InvalidOperationException` if no session exists
   - `ISessionStore` and `TokenRefreshService` keep JWT alive during MCP server lifetime
   - No env var credentials / no Otp.NET

5. **Error handling:**
   - Per-category error isolation in `finary_get_all_accounts` and `finary_get_all_transactions`
   - Bootstrap validation: all data tools check org context is set
   - Transaction category filtering: `finary_get_transactions_by_category` validates `HasTransactions()` with clear error messages

6. **Tool signatures:**
   - All tools use `[McpServerTool(Name = "...")]` with `[Description]` on methods + parameters
   - Tool names are snake_case, descriptions are LLM-facing
   - Parameters: strictly typed (enums, strings, ints)
